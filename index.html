🛰️ Thème — Localisation GPS

Le GPS (Global Positioning System) détermine la position d'un point sur Terre grâce à des signaux émis par des satellites. En mesurant le temps de trajet du signal de 3 satellites au minimum, on calcule les coordonnées (latitude, longitude) par triangulation.

Coordonnées GPS : latitude (Nord/Sud, entre -90° et +90°) et longitude (Est/Ouest, entre -180° et +180°).

✅ Correction 1.1

📷 Exercice 1.2 — Encoder la photo de l'incendie
📚 Rappel — Pixel & Compression

Une image numérique est une grille de pixels. Chaque pixel possède une valeur de couleur (ex. 0 = blanc, 1 = gris, 2 = noir pour 2 bits). La compression RLE (Run-Length Encoding) encode les séquences répétées : 5B 3N = 5 blancs puis 3 noirs.

Voici la photo 8×8 de la zone incendie, déjà partiellement complétée. Chaque case = 1 pixel. Code : B=Blanc, G=Gris, N=Noir.

Grille 8×8 — Photo zone incendie (à compléter sur papier)

Cliquez sur une case pour la faire basculer Blanc → Gris → Noir (simulation écran seulement)

La grille modèle à reproduire sur papier (lecture ligne par ligne) :

L1: B B B B B B B B
L2: B B B G G B B B
L3: B B G N N G B B
L4: B G N N N N G B
L5: B G N N N N G B
L6: B B G N N G B B

L8: B B B B B B B B

  2. Encode les lignes L3 et L4 en compression RLE.
  3. Combien de pixels contient cette image ? Combien de bits faut-il si on code chaque pixel sur 2 bits ?
  4. Quel objet la photo représente-t-elle ? (indice : regarde la forme globale)
✅ Correction 1.2
  1. Grille complétée selon le modèle. (vérification visuelle par l'enseignant·e)

🌐 Thème — Internet & Protocole TCP/IP

Sur Internet, les données sont découpées en paquets. Chaque paquet contient : une adresse source, une adresse destination, un numéro d'ordre et une charge utile (les données). Les routeurs acheminent les paquets de proche en proche. Le protocole TCP garantit la réception de tous les paquets et demande le renvoi des paquets perdus.

✅ Correction 2.1
  1. Chemin A : R1 → R2 → R3 → R4 (3 sauts) — Chemin B : R1 → R2 → R6 → R7 → R4 (4 sauts)
  2. Chemin A : 10 + 8 + 12 = 30 ms. C'est le plus rapide.
  3. Internet est un réseau maillé : si un lien tombe, les paquets empruntent automatiquement un autre chemin. C'est le principe de tolérance aux pannes.
🎭 Jeu de Rôle — Simulation TCP/IP (toute la classe)

Rôles :

  • 1 élève = Source R1 (envoie les paquets)
  • 1 élève = Destination R4 (pompiers)
  • Les autres = Routeurs R2, R3, R5, R6, R7

Déroulement :

  1. R1 découpe le message d'alerte en 3 paquets (boulettes de papier numérotées 1, 2, 3). Chaque paquet contient : n°, source, destination, message.
  2. Chaque routeur reçoit le paquet et le transmet au routeur suivant selon la table de routage ci-dessous.
  3. L'enseignant·e peut "couper" un lien (retirer le routeur du jeu) pour simuler une panne : les routeurs doivent contourner.
  4. Si un paquet arrive abîmé (froissé), R4 demande le renvoi (simulation ACK/NACK).
  5. R4 reconstitue le message dans l'ordre des numéros.

🌐 Thème — Le Web & HTML

Le Web est un service d'Internet permettant d'accéder à des pages via un navigateur. Les pages sont écrites en HTML (HyperText Markup Language) avec des balises qui structurent le contenu. Exemple : <h1>Titre</h1>, <p>paragraphe</p>, <a href="...">lien</a>.

🖥️ Exercice 3.1 — La page Web d'alerte (sur papier)

Construis sur une feuille A4 la maquette d'une page HTML d'alerte incendie. Utilise des post-its pour les balises. Chaque post-it = 1 balise avec son contenu.

📋 Structure à reproduire
<html>
  <head>
    <title>Alerte Incendie - Numéropolis</title>
  </head>
  <body>
    <h1>🔥 ALERTE INCENDIE</h1>
    <p>Localisation : Case C7 — Quartier Nord</p>
    <p>Heure : 14h32 — Niveau : CRITIQUE</p>
    <img src="incendie.jpg" alt="Photo zone">
    <a href="pompiers.html">Contacter les pompiers</a>
  </body>
</html>
  1. Reproduis cette structure sur papier avec des post-its (1 balise = 1 post-it coloré).

  3. Ajoute une balise <table> contenant un tableau à 2 colonnes : "Paramètre" et "Valeur" (avec 3 lignes de données de ton choix).
✅ Correction 3.1

  2. La balise <a> crée un hyperlien. L'attribut href indique l'URL de destination (adresse de la page cible).
  3. Exemple : <table><tr><th>Paramètre</th><th>Valeur</th></tr><tr><td>Température</td><td>450°C</td></tr>...</table>
📊 Thème — Données & Format CSV

Un fichier CSV (Comma-Separated Values) stocke des données tabulaires sous forme de texte. Chaque ligne est un enregistrement, les champs sont séparés par des virgules ou des points-virgules. Exemple : nom,latitude,longitude,niveau

📱 Thème — Réseaux Sociaux & Graphes

Un réseau social peut être modélisé par un graphe où chaque personne est un nœud et chaque relation d'amitié/abonnement une arête. La centralité de degré d'un nœud = nombre de connexions directes. Plus elle est élevée, plus l'information se propage vite depuis ce nœud.

🕸️ Exercice 3.3 — Le réseau social de Numéropolis

Voici le graphe du réseau social urbain. Les nœuds sont des habitants, les arêtes des abonnements. Une alerte est postée par Alice.

  1. Calcule le degré de chaque nœud (nombre de liaisons directes). Qui a le plus fort degré ?

  3. Si Bob quitte le réseau (on supprime son nœud), le réseau reste-t-il connexe ? Justifie.
  4. Calcule le nombre total d'arêtes dans ce graphe.
✅ Correction 3.3

  2. Maximum 2 étapes : Alice → Bob/Clara/David (1 étape) → Eva/Frank/Gina/Hugo (2 étapes).

🤖 Thème — Systèmes Embarqués & IoT

Un système embarqué est un ordinateur miniaturisé intégré dans un objet pour accomplir une tâche spécifique. Exemples : détecteur de fumée connecté, capteur de température, caméra de surveillance. L'ensemble des objets connectés forme l'Internet des Objets (IoT — Internet of Things). Ces systèmes collectent des données, les transmettent et peuvent déclencher des actions automatiques.

⚖️ Exercice 4.2 — Débat Citoyen : La ville intelligente

La ville de Numéropolis envisage de déployer 500 capteurs supplémentaires, dont des caméras à reconnaissance faciale et des capteurs de bruit dans les espaces publics.

✅ Arguments POUR
  • Réponse plus rapide aux urgences
  • Prévention des accidents
  • Optimisation des transports
  • Économies d'énergie
⚠️ Arguments CONTRE
  • Surveillance de masse
  • Atteinte à la vie privée
  • Biais algorithmiques
  • Données piratables
  1. Débat oral (5 min) : La moitié du groupe défend le projet, l'autre s'y oppose. L'enseignant·e chronomètre.
  2. Qui a accès aux données de géolocalisation collectées ? Qui devrait y avoir accès ? Justifie.
  3. Qu'est-ce que le RGPD ? Comment s'applique-t-il ici ?
  4. Propose une solution technique qui permettrait de bénéficier des avantages tout en protégeant la vie privée.
💡 Pour aller plus loin

Le RGPD (Règlement Général sur la Protection des Données, 2018) oblige à collecter seulement les données nécessaires (minimisation), à informer les personnes concernées et à obtenir leur consentement. Les données biométriques (visage) sont une catégorie sensible soumise à des règles strictes.

✅ Éléments de réponse 4.2
  1. Débat oral — évaluation formative par l'enseignant·e.
  2. Actuellement : la mairie, les opérateurs, les fournisseurs de services. Idéalement : uniquement les services d'urgence avec accès limité et traçable. Les citoyens devraient pouvoir accéder à leurs propres données.
  3. Le RGPD impose : consentement, finalité limitée, durée de conservation définie, droit d'accès/suppression. Ici : obligation d'afficher des panneaux, de limiter la durée de stockage des images.
  4. Solutions : anonymisation des données dès la collecte (flou sur les visages), traitement local (edge computing) sans envoi au cloud, chiffrement end-to-end, audit indépendant.
